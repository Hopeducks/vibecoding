## 🤖 생성형 AI 이용 시 기본 지침

> 생성형 AI 도구(ChatGPT, Claude, Gemini 등)를 사용하실 때 입력해두면 더 신뢰할 수 있는 답변을 받을 수 있습니다.

### ✅ 전체 지침 (한 번에 복사)

```
생성형 AI 사용 시 기본 지침

1. 충분한 근거가 없거나 정보가 불확실한 경우, 임의로 지어내지 말고 "알 수 없습니다" 또는 "잘 모르겠습니다"라고 명시하고 그렇게 판단한 근거를 알려주세요.

2. 답변하기 전, 단계별로 가능한 정보를 모두 검증하고, 모호하거나 출처가 불분명한 부분은 "확실하지 않음"으로 표시하세요.

3. 최종적으로 확실한 정보만 사용하여 답변을 완성하세요. 만약 추측이 불가피할 경우, 이에 대한 부분을 명시해주세요.

4. 사용자의 요청이나 문의가 모호하거나 추가 정보가 필요하다고 판단되면, 사용자에게 맥락과 세부정보를 더 요청하세요.

5. 확인되지 않은 사실에 대해 확신을 갖고 단정하지 말고 필요한 경우 근거와 함께 추론하세요.

6. 각 답변에 대해 출처나 근거가 있다면 해당 정보를 명시하고, 가능하면 관련 링크나 참고자료를 요약해 함께 제시하세요.
```

## 🏫 교육용 웹앱 만들기 실습

> **이 실습에 대하여**
>
> 생성형 AI(Claude, ChatGPT 등)를 활용하여 교과 단원과 연계된 **교육용 웹 애플리케이션**을 직접 설계하고 제작하는 실습입니다.  
> 프로그래밍 경험이 없어도 프롬프트를 단계별로 입력하는 것만으로 완성도 있는 웹앱을 만들 수 있습니다.
>
> **아래 예시는** 초등학교 5학년 과학 교과 **"용해와 용액"** 단원을 중심으로 제작한 샘플 프롬프트입니다.  
> `[  ]` 표시된 부분을 본인의 교과·학년·단원으로 바꾸어 활용하세요.

---

### 0단계 — AI에게 역할 부여하기

> 💡 **왜 필요한가?**  
> 대화를 시작하기 전에 AI에게 "교육 전문가" 역할을 부여하세요.  
> 이후 모든 답변의 맥락이 교육적 관점에서 일관되게 유지됩니다.  
> 매우 간단하지만 결과물의 품질을 크게 높이는 핵심 단계라고 할 수 있습니다.

```
당신은 초등학교 교육과정과 에듀테크(EduTech)에 정통한 교육 전문가이자 웹 개발자입니다.
2022 개정 교육과정을 정확히 이해하고 있으며, 학습자 수준에 맞는 콘텐츠 설계와
인터랙티브 학습 도구 제작에 풍부한 경험을 가지고 있습니다.
앞으로 제가 요청하는 교육용 웹 애플리케이션 제작을 함께 진행해 주세요.
```

---

### 1단계 — 교육용 웹앱 뼈대 만들기

#### [1-1] PRD(제품 요구사항 문서) 작성 요청

> 💡 **왜 필요한가?**  
> PRD는 설계도입니다. AI에게 바로 "코드 짜줘"라고 하면 방향이 흔들리기 쉽습니다.  
> 먼저 PRD를 받아 검토하고 피드백한 뒤 다음 단계로 넘어가세요.

```
초등학교 교육용 웹 애플리케이션을 제작하려고 합니다.

[기본 정보]
- 대상 학년: 초등학교 5학년
- 교과: 과학
- 단원: 용해와 용액 (2022 개정 교육과정)
- 목적: 단원 학습 내용 복습 및 자기 평가

[핵심 기능]
1. 실험 시뮬레이션: 학생들이 실제 수업에서 진행한 실험(용질이 물에 녹는 과정, 용액의 농도 비교 등)을 인터랙티브하게 재현할 수 있는 기능
2. 퀴즈(4문항): 2022 개정 교육과정의 성취기준을 달성했는지 스스로 점검할 수 있는 퀴즈. 단순 암기가 아닌 개념 이해와 탐구 능력을 평가할 수 있도록 설계
3. 학습 피드백: 퀴즈 결과에 따라 격려 메시지 또는 복습 안내를 제공

[기술 요구사항]
- 별도 설치 없이 웹 브라우저에서 바로 실행 가능한 단일 HTML 파일
- 태블릿·스마트폰 등 다양한 기기에서 정상 작동 (반응형 디자인)
- 외부 서버 연동 없이 완전히 동작 (인터넷 연결 불필요한 구조 권장)

[교육적 요구사항]
- 모든 텍스트와 설명은 초등 5학년 수준의 어휘와 문장으로 작성
- 과학적으로 정확한 내용만 포함 (오개념 유발 요소 배제)
- 학습자가 스스로 탐색하고 답을 발견하는 구성

위 내용을 바탕으로 구현 방향과 함께 구체적인 PRD를 작성해 주세요.
```

---

#### [1-2] HTML 기본 구조(뼈대) 생성 요청

> 💡 **왜 필요한가?**  
> 기능은 나중에 붙입니다.
> 먼저 전체 레이아웃의 "뼈대"를 확인하고 구조를 승인한 뒤 2단계로 넘어가야 이후 수정이 쉬워집니다.

```
작성된 PRD대로 진행하겠습니다.
PRD를 기반으로 HTML 문서의 기본 구조를 작성해 주세요.

[작성 조건]
- 헤더, 시뮬레이션 섹션, 퀴즈 섹션, 결과/피드백 섹션, 푸터로 구성
- 각 섹션에는 나중에 수정하기 쉽도록 고유한 id 이름을 붙일 것
  (예: id="section-simulation", id="section-quiz")
- 이 단계에서는 CSS 디자인이나 JavaScript 기능은 구현하지 않을 것
- 각 섹션 안에 어떤 내용이 들어갈지 주석(<!-- -->)으로 표시할 것
- 한국어 인코딩(UTF-8)을 선언하고, 반응형을 위한 viewport 메타태그를 포함할 것
```

### 2단계 — 웹앱 기능 부여하기

> 💡 **이 단계는 실습자 본인이 자유롭게 작성합니다**  
> 뼈대를 확인한 후, 넣고 싶은 기능을 직접 프롬프트로 요청하세요.  
> 아래는 참고용 예시입니다.

**예시 A — 실험 시뮬레이션 기능 추가**
```
시뮬레이션 섹션(#section-simulation)에 기능을 구현해 주세요.

[구현할 내용]
- 사용자가 슬라이더로 "용질의 양"을 조절하면 비커 안 용액의 색깔과 투명도가 변하는 인터랙션
- 용질이 완전히 녹은 상태와 더 이상 녹지 않는 포화 상태를 시각적으로 구분
- 각 상태에 어울리는 설명 텍스트가 자동으로 표시되도록 구성
- JavaScript와 CSS 애니메이션만 사용하고 외부 라이브러리는 최소화
```

**예시 B — 퀴즈 기능 추가**
```
퀴즈 섹션(#section-quiz)에 기능을 구현해 주세요.

[구현할 내용]
- 4지선다 객관식 문항 4개
- 문항은 2022 개정 교육과정 "용해와 용액" 단원의 성취기준을 반영
- 정답 선택 시 즉각 피드백 (맞았을 때/틀렸을 때 다른 색상과 메시지)
- 4문항 완료 후 점수 합산 및 수준별 격려 메시지 출력
  · 4점: "완벽해요! 진정한 과학자네요 🎉"
  · 3점: "잘했어요! 한 문제만 더 확인해봐요 😊"
  · 2점 이하: "다시 한번 시뮬레이션을 해보고 도전해봐요 💪"
- 다시 풀기 버튼 포함
```

**예시 C — 학습 진행 상태 저장 기능**
```
학생이 페이지를 새로 고침해도 퀴즈 진행 상태가 유지되도록
브라우저의 localStorage를 활용한 상태 저장 기능을 추가해 주세요.
저장되는 항목: 퀴즈 점수, 마지막으로 푼 문항 번호
```

### 3단계 — 웹앱 디자인하기

> 💡 **레퍼런스를 함께 제시할수록 결과가 좋아집니다**  
> 마음에 드는 웹사이트 링크, 색상 코드, 키워드를 함께 넣어보세요.

```
완성된 HTML 문서에 디자인을 적용해 주세요.

[디자인 방향]
- 초등학교 5학년 학생이 "진짜 과학자가 되어 실험하는 느낌"이 나도록 구성
- 실험실·과학 탐구 분위기를 표현하되, 어둡거나 무겁지 않고 밝고 호기심 넘치는 톤
- 모던하고 세련된 UI이면서도 아이들이 직관적으로 사용할 수 있는 UX

[구체적인 요구사항]
- 색상: 물·용액을 연상시키는 파란색·청록색 계열을 메인으로, 포인트 컬러는 밝은 노랑 또는 주황
- 폰트: 가독성이 높은 한글 폰트 (Google Fonts의 Noto Sans KR 또는 Nanum Gothic)
- 버튼과 카드에 부드러운 hover 애니메이션 적용
- 시뮬레이션 영역에 물방울·거품 등 과학적 요소를 시각적으로 표현
- 모바일·태블릿에서도 레이아웃이 깨지지 않도록 반응형 처리

[참고 레퍼런스]
(여기에 마음에 드는 웹사이트 주소를 붙여넣으세요)
예) https://www.nationalgeographic.com/education
```

### 4단계 — 웹앱 검토하여 완성도 높이기

#### [4-1] 기능·내용 중심 버그 수정

> 💡 **언제 사용하나?**  
> 웹앱의 기능이 완성된 후, 학생에게 배포하기 전에 결과물을 검토하세요.  
> AI가 스스로 코드 전체를 검토하고 문제점을 찾아 수정합니다.

```
완성된 웹앱 전체를 꼼꼼히 검토하고 수정해 주세요.

[검토 항목]
1. PRD 충족 여부: 처음 요구사항 문서(PRD)의 모든 기능이 빠짐없이 구현되었는지 확인
2. 링크·버튼 동작: 모든 버튼, 링크, 슬라이더가 의도대로 작동하는지 확인
3. 과학적 정확성: "용해와 용액" 관련 설명, 용어, 시뮬레이션 동작이 과학적으로 올바른지 확인
   - 2022 개정 초등 5학년 과학 교육과정 기준에 부합하는지 검토
   - 오개념을 유발할 수 있는 표현이 있다면 수정
4. 학습자 수준 적합성: 어휘, 문장 길이, 설명 방식이 초등 5학년 수준에 맞는지 확인
5. 반응형: 모바일(320px), 태블릿(768px), 데스크톱(1280px) 화면 크기에서 레이아웃이 깨지지 않는지 확인
6. 세부 버그: 위 항목 외에 발견된 모든 버그를 목록으로 정리한 뒤 전면 수정

검토 결과를 항목별로 간략히 보고한 후, 수정된 전체 코드를 제공해 주세요.
```

#### [4-2] UI/UX 고도화 + 탐구심 강화 버전 (선택)

> 💡 **언제 사용하나?**  
> [4-1] 검토를 완료한 후, 완성도를 한 단계 더 높이고 싶을 때 사용하세요.

```
[4-1]에서 버그 수정이 완료된 웹앱을 바탕으로, UI/UX를 전면 고도화해 주세요.

[기능·내용 검토]
- [4-1]과 동일한 기준(PRD 충족, 링크 동작, 과학적 정확성, 학습자 수준)으로 한 번 더 검토
- 미처 수정되지 않은 버그가 있다면 함께 수정

[UI/UX 고도화 방향]
1. 과학적 탐구심 자극
```
